# Эссе

## 1. Почему эта задача — что зацепило лично, для кого решаете
Меня зацепила связка API с алгоритмами оптимизации.
Это реальный продукт, а не очередной чат бот. 
Плюс я сам люблю путешествовать, 
поэтому хотелось сделать инструмент для себя,
который будет помогать с выбором и экономить время.

Это решение для ленивых туристов, которым хочется прогуляться по городу не читая тысячи книг и брошюр,
чтобы пройти по основным достопримечательностям. 
Можно показать пользователю, что не обязательно наматывать лишние километры из-за неправильно построенного маршрута.

## 2. Где агент сломался — 1–2 честных failure case
Я столкнулся с двумя проблемами, которые показывают ограничения моделей:

**Случай 1: галлюцинации при отсутствии данных.**
Изначально парсер OpenStreetMap был настроен только на поиск точек.
Когда агент искал крупные объекты, например, торговые центры, 
которые размечены полигонами, то API возвращал пустоту. 
Чтобы не расстраивать пользователя, агент с температурой, не равной нулю по умолчанию,
начинал галлюцинировать и выдумывал несуществующие адреса, названия мест или координаты.
Пришлось сделать `temperature=0.0` и добавить строгие ограничения в системный промпт
и научить скрипт парсить здания (nwr).

**Случай 2: потеря контекста при передаче массивов.**
Агент успешно получал JSON с местами от инструмента поиска,
но когда ему нужно было передать этот массив в инструмент `optimize_route`,
то модель ленилась, и вместо передачи реальных данных, она отправляла пустой массив `[]`,
получала дистанцию 0.0 км, а в итоговом ответе выдумывала несуществующие места.
Решением стало снижение нагрузки на модель и переход на более мощную модель `llama-3.3-70b-versatile`:
я ограничил выдачу API до топ-3 мест и прописал в промпте команду не терять переменные между вызовами функций
и стал использовать модель с большим числом параметров.

## 3. Что сделали бы дальше при наличии времени
Если бы было больше времени я бы сделал: 
1. поиск точного адреса места и его описания, а не только названия и координат
2. поиск местоположения пользователя по фото или построение маршрута до этого места от пользователя
3. сделать веб интерфейс для визуализации пути на картах
